# MTM Workshop Computer VCV Rack Plugin

A VCV Rack 2 plugin for the [Music Thing Modular Workshop Computer](https://www.musicthing.co.uk/pages/workshop-computer) вЂ” an 8HP programmable Eurorack module based on the RP2040 (Raspberry Pi Pico).

This plugin compiles and runs the original Workshop Computer firmware card source code directly inside VCV Rack, using a complete Pico SDK hardware emulation layer.

## Features
- **~55 program cards** вЂ” all official releases plus external community cards
- **Dual-core emulation** вЂ” Core1 background threads via `std::thread`
- **Flash memory** вЂ” 2MB simulated flash, persisted per card
- **MIDI I/O** вЂ” Full MIDI routing to/from card programs
- **Monome Grid** вЂ” Virtual grid controller support via monome-rack
- **Web UI** вЂ” Embedded HTTP/WebSocket server for card manager UIs
- **USB Audio Bridge** вЂ” Audio routing for the USB Audio card

## Building

### Prerequisites
- VCV Rack 2 SDK in `deps/Rack-SDK`
- Clang/LLVM on macOS (Xcode command line tools)

### Clone with submodule
```bash
git clone --recursive https://github.com/vincentltm/Workshop_Computer_VCV.git
cd Workshop_Computer_VCV
# SDK goes in deps/Rack-SDK (download from VCV Rack website)
make -j8
make install-dev
```

### Update firmware submodule (deliberately)
```bash
# Stay on pinned version:
git submodule update

# Pull latest from upstream (only when you want to):
git -C deps/Workshop_Computer pull origin main
git add deps/Workshop_Computer
git commit -m "bump Workshop_Computer submodule"
```

### Used as a submodule (by Workshop_System_VCV)
This repo is designed to be included as a git submodule inside the
[Workshop_System_VCV](https://github.com/vincentltm/Workshop_System_VCV) repo.
The `src/shared/ComputerWidgets.hpp` header is consumed by both plugins to ensure
the card slot UI is always identical between the standalone and integrated versions.

## Architecture Notes

### IComputerModule interface
All shared widgets (`ProgramCardWidget`, `WorkshopToggleSwitch`, `WorkshopResetButton`)
talk to the hosting module through `IComputerModule` in `src/shared/ComputerWidgets.hpp`.
This allows them to be used identically in both `WorkshopComputer` and `WorkshopSystem`.

### Flash storage
Flash `.bin` files are stored next to the plugin's installed location using
`rack::asset::plugin()` вЂ” no hardcoded paths.

### Card loading
Each card is compiled as an independent shared library (`res/cards/libcard_*.dylib`).
At runtime, cards are `dlopen`'d with a unique `install_name_tool` identity to allow
multiple instances without shared static variables.

### Adding new cards
1. Add card source to `deps/Workshop_Computer/releases/` or `deps/external/`
2. Run `python3 tools/port_all_cards.py` to regenerate `src/cards/` and `Makefile.cards`
3. `make -j8` to build
